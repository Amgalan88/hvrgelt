# hvrgelt — Supabase тохиргооны заавар

Энэ апп одоо Supabase (өгөгдлийн сан) дээр ажиллахаар бэлэн боллоо.
Доорх 4 алхмыг дагаснаар бүх захиалга, бүртгэл бодитоор хадгалагдаж,
жолооч/оператор хооронд real-time хуваалцагдана.

---

## 1. Supabase project үүсгэх

1. https://supabase.com → **Sign in** (GitHub/Google-ээр үнэгүй).
2. **New project** дарна.
   - Name: `hvrgelt`
   - Database Password: хүчтэй нууц үг (хадгалж аваарай)
   - Region: `Southeast Asia (Singapore)` (Монголд хамгийн ойр)
3. Project үүсэхийг ~2 минут хүлээнэ.

## 2. Хүснэгтүүдийг үүсгэх (SQL)

1. Зүүн талын цэснээс **SQL Editor** → **New query**.
2. Төслийн дотор байгаа [supabase_schema.sql](supabase_schema.sql) файлын
   **бүх агуулгыг** хуулж, query цонхонд буулгана.
3. **Run** (эсвэл Ctrl+Enter) дарна.
4. "Success" гарвал хүснэгт + анхны өгөгдөл орсон гэсэн үг.

## 3. API түлхүүрээ авах

1. Зүүн доод талын **Project Settings** (араат дугуй) → **API**.
2. Дараах 2 утгыг хуулж ав:
   - **Project URL** (жишээ: `https://abcdxyz.supabase.co`)
   - **anon public** key (`eyJ...` гэж эхэлдэг урт түлхүүр)

## 4. .env файл үүсгэх

1. Төслийн үндсэн фолдерт `.env.example`-г хуулж `.env` нэртэй болго.
2. Дотор нь дээр авсан 2 утгаа бөглө:

```
VITE_SUPABASE_URL=https://abcdxyz.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOi... (anon public key)
```

3. Хадгална. (`.env` нь git-д орохгүй — нууц хэвээр үлдэнэ.)

## Ажиллуулах

```
npm run dev
```

Браузерт нээгдсэн линкээр орж туршина.

### Туршилтын нэвтрэх мэдээлэл (seed дата)

| Үүрэг       | Утас      | Нэвтрэх          |
|-------------|-----------|------------------|
| Супер админ | 99739959  | нууц үг: gegee0011 |
| Оператор    | 99110001  | PIN: 1234        |
| Жолооч      | 99112233  | PIN: 1111        |
| Захиалагч   | шинээр бүртгүүлнэ | —          |

### Шалгах зүйл
- Захиалга үүсгээд хуудсаа **refresh** хий → захиалга **арилахгүй** байх ёстой.
- Хоёр өөр браузер (эсвэл утас + комьпютер) дээр оператор + жолоочоор нэвтэрвэл
  нэг нь захиалга томилоход нөгөөд нь **шууд (real-time)** гарч ирнэ.

## ⚠️ Аюулгүй байдлын анхааруулга (чухал)

Одоогийн тохиргоо нь **демо/тест** зориулалттай:

- PIN/нууц үг өгөгдлийн санд **задгай текстээр** хадгалагдаж байгаа.
- RLS policy нь хэн ч (anon) бүх өгөгдлийг унших/бичих эрхтэй.

**Бодит хэрэглээнд гаргахаасаа өмнө** дараахыг хийх шаардлагатай:
1. Supabase Auth (жинхэнэ нэвтрэлт) рүү шилжих.
2. Нууц үгийг hash хийх.
3. RLS policy-г үүрэг тус бүрээр хязгаарлах (зөвхөн өөрийн өгөгдлийг харах гэх мэт).

Эдгээрийг хийх бол надад хэлээрэй.
